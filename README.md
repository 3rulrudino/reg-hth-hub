# reg-hth-hub

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件形式存储，可直接通过 GitHub Pages 或其他静态托管服务访问。

## 项目简介

reg-hth-hub 是一个通用型的 HTML 页面归档仓库。它不针对任何特定域名或网站，而是作为一组独立页面的集中存放与发布点。每个页面可视为一个自包含的功能单元，适用于展示、工具、信息页等场景。

## 目录结构

```
.
├── index.html          # 首页 / 导航页
├── pages/              # 存放各个独立 HTML 页面
│   ├── page-1.html
│   ├── page-2.html
│   └── ...
└── assets/             # 公共资源文件（样式、脚本、图片等）
    ├── css/
    ├── js/
    └── images/
```

- `index.html`：仓库入口页面，通常包含页面列表或导航链接。
- `pages/`：每个子页面为一个独立的 HTML 文件，可单独访问。
- `assets/`：存放所有页面可能共用的静态资源，便于统一管理。

## 页面归档说明

- 所有页面均为静态 HTML，不依赖后端服务。
- 页面之间彼此独立，互不干扰。
- 页面可引用 `assets/` 下的公共资源，也可内联自身所需的样式与脚本。
- 建议每个页面保持自包含性，减少外部依赖，便于长期归档与迁移。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新的 HTML 文件，并更新 `index.html` 中的导航链接（如有）。
- 更新页面：直接修改对应 HTML 文件内容。
- 删除页面：移除文件并清理 `index.html` 中的相关链接。
- 公共资源变更：修改 `assets/` 下对应文件，注意检查是否影响已有页面。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 中添加或修改 HTML 文件。
3. 可通过本地直接打开 HTML 文件预览，或部署至静态托管服务后访问。
